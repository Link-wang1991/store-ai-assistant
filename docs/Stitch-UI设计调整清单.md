## 一、整体判断

当前稿件有以下优点，应保留：

- 整体已经转向轻量、白色、细边框和克制绿色，比传统 CRM 后台更接近目标方向。
- 首页已经具备“概览卡 + AI 工作项”的 Inbox 结构。
- 会谈页具备录音入口和会谈处理状态。
- 客户详情已出现今日简报、AI 画像、AI 记忆和互动时间线。
- AI 教练已出现关联客户、参考资料和回答反馈。
- 管理中心开始把风险、知识缺口和未完成动作放在前面。
- 已经吸收了参考图的部分视觉 DNA：宽松留白、圆角卡片、模块化数据区、搜索/语音入口和轻量状态色。

但当前六张稿子仍像来自不同设计系统，且部分业务逻辑与产品定义冲突。下一轮不要继续增加装饰，而应优先解决：

1. 页面和组件视觉不统一；
2. 示例数据与业务状态矛盾；
3. AI 信息过长，缺少行动闭环；
4. 同名客户和客户身份识别不足；
5. 多个按钮看起来可点击，但动作含义不清；
6. 空状态、加载状态、失败重试和降级状态没有系统设计。

## 参考图的正确转译方式

这张参考图是桌面金融 Dashboard，适合参考“视觉秩序”，不适合直接参考“业务组件”。Stitch 下一轮需要保留其气质，同时完成以下移动端和业务转译：

| 参考图中的设计语言 | 在门店 AI 助手中的正确转译 |
| --- | --- |
| 大面积白色、宽松留白 | 保留，但移动端要控制首屏高度，不能让关键任务被推到屏幕下方 |
| 多尺寸模块化卡片 | 转成纵向优先的 AI Inbox 卡片，不做金融仪表盘式拼图 |
| 顶部助手问候 | 改成今日经营摘要，不做“需要帮忙吗”的通用聊天问候 |
| 搜索与语音入口 | 搜索用于客户/会谈/知识/任务；语音入口必须明确进入 AI 教练或会谈，不能只是装饰 |
| Financial Dashboard 指标卡 | 转成今日到店、待跟进、待复盘和风险等可行动指标 |
| Activity Manager | 转成“今日 AI 工作台”，按经营价值和风险排序 |
| 橙红强调色 | 替换为产品现有的克制绿色；橙色仅用于待处理，红色仅用于风险 |
| 图表和进度环 | 只在有真实趋势或完成率时使用，不为装饰添加虚荣指标 |
| 大圆角、轻阴影 | 保留，但卡片层级统一，避免每一块都像独立悬浮窗口 |
| 桌面横向信息密度 | 移动端改为“动作优先、依据折叠”的纵向渐进式展示 |

因此，当前设计不需要推翻参考风格。需要做的是：

1. 保留参考图的高级、轻盈、模块化和留白；
2. 删除从金融 Dashboard 机械迁移过来的通用问候、装饰性语音按钮、虚荣指标和大块宣传卡；
3. 用客户、会谈、任务、风险、知识缺口和 AI 行动替换金融组件；
4. 让所有模块最终指向可执行动作和结果回流。

## 二、必须先统一的全局规范

### 1. 统一产品名称和功能名称

- 顶部品牌统一使用：`门店 AI Inbox`。
- 底部导航固定为：`会谈 / 客户 / 首页 / AI 教练 / 我的`。
- 不要混用“AI 策略教练”“AI 教练”“AI 洞察”等多个主功能名称。
- 主功能统一叫“AI 教练”；“AI 洞察”只能作为某条信息的来源标签。
- “会谈”不要与“会议”“咨询记录”随意混用。
- 管理端如需内部会议，应明确叫“内部会议”；客户沟通统一叫“会谈”。

### 2. 统一画板尺寸和移动端密度

- 所有员工端页面按 390px 宽设计，并验证 375px、390px、430px。
- 当前客户详情稿过窄，文字和控件明显小于其他页面，需要重新按 390px 画板设计。
- 正文字号建议不低于 13–14px，辅助文字不低于 11–12px。
- 主要点击区域不小于 44×44px。
- 底部固定栏、输入栏和内容区之间必须预留安全距离，不能遮挡最后一条内容。

### 3. 统一视觉系统

- 页面主背景使用极浅灰或接近白色，不要每个页面都铺大面积浅绿色。
- 绿色只用于 AI、主操作、已完成和重点强调，不作为所有区域的背景。
- 统一卡片圆角、边框、阴影和内边距，避免一页 12px 圆角、一页 24px 圆角。
- 统一图标风格，全部使用同一套线性图标，不混用实心、手绘、装饰性图标。
- 头像统一策略：有真实员工/客户头像时使用照片，没有时使用姓名首字；不要混用不相关的外国人物素材。
- 不使用中文斜体长段落，降低阅读效率。
- 删除类似“绿色未读圆点”的装饰，状态使用标签、文字和细色条表达。

### 4. 统一示例数据

- 所有页面使用同一组中文门店、员工、客户和项目示例。
- 不再混用 `Elena Rodriguez`、`AM/PM`、英文会员标签和中国门店场景。
- 日期统一为中文格式，例如“今天 10:30”“10 月 24 日”。
- 示例客户必须跨页面保持一致：客户身份、到店次数、项目、负责人和当前状态不能互相冲突。

### 5. 统一信息层级

页面信息优先级固定为：

```text
当前要处理的动作
→ 为什么要处理 / AI 判断
→ 可以直接使用的话术
→ 风险和升级条件
→ 判断依据与历史详情
```

- 首屏不要先展示长篇 AI 分析。
- 依据、来源、完整历史默认折叠。
- 一张卡片只保留一个主要动作，其他操作降低层级。

### 6. 统一 AI 标识与可信度

所有 AI 内容统一显示以下信息：

- 内容类型：门店事实 / 客户沟通建议 / 通用建议 / 待确认 / 高风险；
- 来源：客户记录、会谈、门店知识库或系统专业知识；
- 更新时间；
- 可信度或“待确认”状态；
- 查看依据、纠错和撤回入口。

不要只用一枚“AI 洞察”胶囊代表所有信息。

### 7. 统一按钮文案

- 避免“执行操作”“分析”“忽略”“立即同步”等含义不明确的按钮。
- 按钮直接说明结果，例如：
  - `查看客户并跟进`
  - `复制回访话术`
  - `记录处理结果`
  - `开始会谈`
  - `重试转写`
  - `提交店长确认`
  - `查看经营明细`
- 高风险卡片不能提供无条件“一键发送致歉微信”。
- “忽略”改为“稍后处理”或“不适用”，并要求选择原因。

## 三、页面 1：会谈页调整

### 当前存在的问题

1. 已选择的客户显示“历史到店 12 次、VIP”，但下方却选中“新客户”，业务逻辑冲突。
2. “新客户 / 老客户”被写成“会谈场景”，实际它们是客户类型，不是会谈场景。
3. 真正的会谈场景缺失，例如新客咨询、护理服务、复购回访、投诉/异常。
4. 客户识别信息不足，遇到多个同名客户时无法确认选中了谁。
5. 缺少录音隐私告知和客户同意状态。
6. “AI 分析中”使用进度条，容易让用户误以为是可精确计算的处理百分比。
7. 失败会谈只显示失败原因，没有“重试转写 / 手动记录 / 重新录音”动作。
8. 近期会谈卡片高度和装饰不一致，第一张紫色光环过重。
9. 页面标题颜色过浅，顶部留白过大，整体不像与其他页面同一套框架。

### 调整要求

会谈设置区改为以下顺序：

1. 客户类型：新客户 / 已有客户；
2. 客户选择：已有客户时显示搜索和识别卡；
3. 会谈场景：新客咨询 / 护理服务 / 成交沟通 / 复购回访 / 投诉与异常；
4. 隐私状态：已向客户告知并获得同意；
5. 主按钮：开始录音。

客户识别卡至少显示：

- 姓名；
- 手机尾号或微信昵称；
- 今日预约时间；
- 最近到店；
- 负责人；
- 最近事项。

会谈状态统一为步骤状态：

```text
录音中 → 上传中 → 转写中 → 分析中 → 已完成
```

失败状态必须带可执行动作：

- 重新上传；
- 重试转写；
- 手动补充；
- 联系负责人。

## 四、页面 2：客户详情页调整

### 当前存在的问题

1. 页面画板过窄，文字、标签和点击区域明显过小。
2. 使用外国客户姓名和素材，与其他页面和实际门店场景不一致。
3. 首屏只有姓名和会员等级，没有足够的同名客户识别信息。
4. “今日简报”方向正确，但缺少未完成承诺、开场话术、重点追问和风险提醒的清晰分层。
5. 客户画像用 85% 进度条表达整体可信度，容易形成虚假精确感。
6. AI 记忆只有彩色竖线，缺少来源、时间、可信度、状态和纠错入口。
7. 互动时间线过密，关键信息不突出。
8. 页面底部同时存在 AI 输入栏和五栏导航，占用空间并可能遮挡内容。
9. 缺少当前任务、增长机会和下一步行动。
10. 页面浅绿色面积过大，缺少纯白、克制的 Inbox 质感。

### 调整要求

首屏模块顺序建议：

1. 客户识别区；
2. 今日服务简报；
3. 当前行动卡；
4. AI 客户画像；
5. AI 记忆；
6. 会谈记录；
7. 互动时间线；
8. 基础资料，默认折叠。

今日服务简报固定展示：

- 最近一次服务和结果；
- 本次可能的核心需求；
- 未完成事项或历史承诺；
- 一句开场话术；
- 三个重点追问；
- 风险、禁忌或注意事项。

AI 记忆每条显示：

```text
内容类别 + 内容
来源 + 时间 + 可信状态
纠错 / 确认为事实 / 撤回
```

底部 AI 快捷提问可以保留，但需要：

- 缩小高度；
- 明确显示当前绑定客户；
- 与底部导航之间预留安全区；
- 页面滚动到底时不遮挡时间线内容。

## 五、页面 3：首页调整

### 当前做得较好的部分

- 四张概览卡结构清晰。
- 今日优先 / 高价值 / 风险三个标签符合产品方向。
- AI 工作项已经包含处理原因、AI 判断、建议动作和推荐话术。
- 风险卡使用独立颜色，方向合理。

### 当前存在的问题

1. “嘿，需要帮忙吗？”太像通用聊天机器人，不像经营 Inbox。
2. 顶部悬浮麦克风用途不清，容易与会谈录音和 AI 教练语音重复。
3. 搜索框过于泛化，没有说明可搜索客户、会谈、知识、话术和任务。
4. 概览卡没有明确可点击反馈和进入后的目标页面。
5. 工作卡内容偏长，原因、判断、动作、话术之间仍有重复。
6. “执行操作”过于模糊。
7. “忽略”不适合风险或重要经营任务。
8. 高风险卡的“发送致歉微信”属于对外动作，不应一键直接执行。
9. 工作卡缺少负责人、建议时间、完成标准和结果反馈。
10. 底部悬浮“+”不知道是新增客户、任务还是会谈，应删除或明确命名。
11. 卡片的 AI 依据不可查看，缺少来源与可信度入口。

### 调整要求

页头改为业务摘要，例如：

```text
早上好，范范
今天有 4 项跟进、1 项风险需要处理
```

搜索占位文案统一为：

```text
搜索客户、会谈、知识、话术或任务
```

概览卡点击后分别进入：

- 今日重点客户列表；
- 今日待跟进任务；
- 今日到店客户；
- 待复盘会谈。

工作卡推荐结构：

1. 客户与任务标签；
2. 为什么现在处理；
3. AI 判断；
4. 一句建议动作；
5. 一段可直接复制的话术；
6. 负责人、时间、完成标准；
7. 主要按钮 + 次要按钮；
8. 处理后选择最小结果。

按钮必须具体，例如：

- 普通任务：`查看客户并跟进` / `稍后处理`；
- 高风险任务：`查看处理方案` / `交给店长`；
- 已准备外部消息：`预览并确认发送`，不能直接发送。

## 六、页面 4：AI 教练对话页调整

### 当前存在的问题

1. 页面顶部先出现一张“AI 策略教练”答案卡，然后才出现用户问题，时间顺序和页面语义混乱。
2. 关联客户只显示姓名和会员等级，不足以避免同名客户串档。
3. 没有明确区分“客户模式”和“通用模式”。
4. AI 回答是长篇连续文字，并且同一句话重复多次，无法现场使用。
5. 长段中文使用斜体，阅读体验较差。
6. 回答没有固定展示“说什么、问什么、做什么、风险”。
7. 对比、行动和风险虽然被单独放出，但缺少负责人、时间、完成标准和结果反馈。
8. “参考资料（2）”没有展示资料名称、有效状态和具体引用依据。
9. 没有标明回答属于门店事实、客户沟通建议还是通用建议。
10. “维持半年以上”等效果表述存在合规风险，若无有效门店知识依据不能直接生成。
11. 点赞、点踩、刷新只有图标，没有清楚说明“有用 / 没用 / 重新生成”。
12. 加号按钮和语音按钮的用途不明确。

### 调整要求

删除顶部重复的“AI 策略教练”大卡，或把它改为明确的“客户服务前简报”，不要与对话答案重复。

顶部上下文区改为：

```text
客户模式
林女士 · 手机尾号 2846 · 今日 15:00 预约 · 负责人小美
[切换客户] [取消关联]
```

通用模式则显示：

```text
通用模式 · 未关联客户
回答基于门店知识与系统专业方法
```

AI 回答首屏固定四个模块：

1. **可以直接说**：一段可复制话术；
2. **接下来要问**：最多 1–3 个问题；
3. **下一步动作**：动作、负责人、建议时间和完成标准；
4. **风险提醒**：风险等级和升级条件。

下方折叠模块：

- 判断依据；
- 参考资料；
- 详细策略；
- 可信度与更新时间。

外部动作必须先预览再确认：

- 生成微信话术；
- 预览消息；
- 确认发送或复制；
- 记录客户结果。

回答底部提供最小结果：

```text
已接受 / 已预约 / 仍有顾虑 / 暂不考虑
未回复 / 需要升级 / 信息有误
```

## 七、页面 5–6：管理中心调整

### 当前做得较好的部分

- 把风险、知识缺口、待处理线索和未完成动作放在首页，符合“管理员只处理例外”的方向。
- 页面整体比传统后台轻量，移动端卡片结构清楚。
- 风险、知识和动作使用了不同但克制的状态色。

### 当前存在的问题

1. “查看所有日志”是系统术语，管理者真正需要的是“查看全部待处理”或“处理记录”。
2. 数量显示为 03、02、08，过于仪表盘化，应直接显示 3、2、8。
3. “紧急 / 培训 / 增长 / 效率”与风险、知识缺口等业务分类混在一起，层级不清。
4. “待处理线索”不是当前产品的稳定业务术语，且“已准备好联系体验”表达含糊。
5. 经营模块只有到店率和 AI 互动率，AI 互动率不等于经营价值，容易成为虚荣指标。
6. “分析”按钮含义不清，应改为查看明细或查看客户。
7. 知识库大面积绿色宣传卡过于抢眼，像营销广告，不像管理任务。
8. 知识库状态写“已更新”，正文又说“需要 AI 同步”，状态互相矛盾。
9. 管理模块使用“会议”，与产品中的客户会谈不一致。
10. 管理中心缺少客户管理、增长动作、通知和权限等明确入口。
11. “今日已达标”等结论缺少数据依据和下钻入口。
